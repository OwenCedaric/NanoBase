III. Tool Arsenal

操作边界与执行原语

BashTool

执行 Shell 命令。对于 Git 提交，优先使用 `/commit` 技能；对于破坏性命令 (reset --hard) 须三思。长时运行任务使用 `run_in_background` 参数而不是 sleep。

src/tools/BashTool/prompt.ts

FileEditTool

精确字符串替换工具。必须先通过 Read 获得内容与行号前缀。编辑时必须保留原始缩进格式，并使用足够独特的 `old_string` 锚点 (通常2-4行相邻代码)。

src/tools/FileEditTool/prompt.ts

EnterPlanModeTool

在面对重大架构重构、不清晰的需求或多文件深远更改时，必须主动使用此工具进入计划模式，与用户探索代码库并取得设计共识，避免无效劳动。

src/tools/EnterPlanModeTool/prompt.ts

EnterWorktreeTool

创建隔离的 Git Worktree 并在其中执行任务。仅当用户明确要求使用 "worktree" 时才调用。适合进行安全且不影响主分支当前状态的独立探索工作。

src/tools/EnterWorktreeTool/prompt.ts

TodoWriteTool

管理结构化任务列表。复杂的多步任务需拆解，记录于待办列表中。任务状态流转：`pending` -> `in_progress` -> `completed`。严格控制仅有一个任务在进行中。

src/tools/TodoWriteTool/prompt.ts

Claude in Chrome
