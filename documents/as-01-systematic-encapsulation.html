Editorial Spec. 01
Architectural Data / System Workflow

Agent
Skill.

它表面上和提示词很接近，
但继续往下拆，会发现它对应的是一套
更完整的任务组织结构。

把资料、案例、结构逐层拆开之后，对 Agent Skill 的关注，更多落在它所体现出的 AI 工作组织方式上。这篇内容将用清晰、可复用的方式，解构其本质。

01. Definition

Agent Skill 是把某一类任务的规则、流程、参考资料、可执行脚本、素材资源，封装成一个可被 AI 按需调用的能力单元。

如果把它类比成人类技能，会比较容易理解。一个厨师会做菜，背后依赖的是一整套能力体系：知道目标菜品是什么、处理顺序、调味规则、在不同场景下怎么调整、该调用什么工具，以及哪些配方和材料要优先使用。

Agent Skill 的思路与此接近。它会把任务所需的规则、资料、流程和资源组织起来，让 AI 在合适的时机完成加载、判断和执行。

02. Vs. Prompts

不仅是
更长的
提示词。

普通提示词更接近一次性的对话指令；Skill 更接近可复用的任务模块。其差异主要体现在三个核心维度：

1

按需加载，拒绝冗余

普通提示词一次性写入大量要求，任务复杂时会导致上下文冗长、信息干扰。Agent Skill 采用分层加载。最上层仅暴露简要描述；模型判断需要时，才读取完整规则、参考资料或执行脚本。内容根据任务需要逐步暴露。

2

解构复杂场景

面对如“品牌物料生成”（含海报、包装、社交配图等多场景），全塞入提示词维护成本极高。Skill 结构将内容拆开组织，按目录约定隔离主说明与附属规则，使任务结构清晰，扩展方便。

3

理解与执行解耦

普通提示词停留在“生成内容”。Skill 则将理解任务和执行动作拆开：模型负责理解，规则文件约束输出，参考资料补充细节，脚本真正执行动作。这让 Skill 更胜任复杂工作流。

03. The Architecture

三层加载机制

以“品牌物料生成”为例。如果只用普通聊天，每次需重复交代品牌名称、调性、视觉风格、输出格式等。采用 Anthropic 常见的 Skill 目录约定，逻辑被清晰划分为三层。

// Standard Directory Structure
SKILL.MD
references/
scripts/
assets/

Layer 1 : Meta Info

让模型知道“技能存在”

最短的描述，位于 SKILL.MD 元信息区，充当目录。告诉模型：技能叫什么、功能范围、触发条件。模型需先知道当前可用技能及其职责。

Layer 2 : Directives

写入具体规则 (SKILL.MD)

当任务匹配时，模型读取 SKILL.MD 主体。这里定义了品牌背景、输出规则（如：默认先输出创意方案，再输出成品建议）、常见交付物。这取代了口语化描述，采用严格配置文件结构。

Layer 3 : Resources

按需调用的底层支撑

references/

补充规则的存放地。按场景拆分（如线下物料、社交媒体规范、包装指南）。模型仅在需要生成特定规格物料时才加载对应文件。

scripts/

负责真正执行动作（如生成海报、调整尺寸）。SKILL.MD 中指引模型传入对应参数调用脚本，将生成与执行剥离。

assets/

统一管理品牌Logo、色板、参考图。约束模型在视觉生成时维持品牌一致性。

04. Application

哪些任务适合做成 Skill？

不是每个人都需要从复杂脚本开始。更合适的起点，是把你最常做、最有标准、最稳定的方法沉淀下来。它必须满足三个条件：

1. 高频重复
2. 有稳定标准
3. 有明确流程

Creator

  • 长文改写
  • Thread 结构生成
  • 文章配图说明生成
  • 固定风格摘要

Educator

  • 教案生成
  • 课件结构生成
  • 练习题设计
  • 知识点讲解框架

Professional

  • 周报整理与会议纪要
  • 项目进展总结
  • 汇报材料初稿

Legal / Audit

  • 风险标注与条款审查
  • 问题归类
  • 审阅建议格式化输出
M V P

与其追求数量，不如
先做一个能用的版本。

如果刚开始接触 Skill，先做 1 个你会高频使用的。它的价值在于持续复用、稳定输出、不断迭代。从最小可用版本开始：

01

先定义任务的边界

明确只负责什么，不负责什么（例如：只改写结构，不负责配图）。

02

先写清核心规则

把最重要的判断规则写明白（保留核心观点、适配节奏、避免过度口语化），让输出稳定。

03

先规范输出格式

格式不明确结果容易飘。规定好先输出什么、再输出什么（如先标题备选，再正文）。

Agent Skill 的价值，体现在它可以让 AI 在明确结构下，稳定调用规则、资料、资源和执行能力。
